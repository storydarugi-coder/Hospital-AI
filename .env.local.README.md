# .env.local 파일 설정 안내

`.env.local` 파일은 로컬 개발 환경에서 사용하는 환경 변수 파일입니다.
이 파일은 개인 API 키가 포함되어 있어 Git에 커밋되지 않습니다.

## 설정 방법

프로젝트 루트에 `.env.local` 파일을 생성하고 다음 내용을 입력하세요:

```bash
# Gemini API 키 (로컬 개발용)
VITE_GEMINI_API_KEY=AIzaSyDOVqA7HP5yRZWalhEu12ECrhqP2R3cetg
```

## 보안 주의사항

⚠️ **중요**: 이 파일은 절대 Git에 커밋하지 마세요!
- `.gitignore`에 이미 `.env.local`이 추가되어 있습니다
- 이 파일에는 개인 API 키가 포함되어 있습니다
- 공개 저장소에 업로드하면 API 키가 노출됩니다

## 이미 생성됨

현재 프로젝트에 `.env.local` 파일이 이미 생성되어 있으며,
새로운 Gemini API 키가 설정되어 있습니다.

로컬 개발 서버를 실행하면 자동으로 이 키가 사용됩니다:

```bash
npm run dev
```

## 다른 개발자를 위한 안내

다른 개발자가 이 프로젝트를 클론한 경우:
1. `.env.example` 파일을 복사하여 `.env.local` 파일을 생성
2. 자신의 Gemini API 키를 `.env.local`에 입력
3. 개발 서버를 실행

```bash
cp .env.example .env.local
# .env.local 파일을 편집하여 본인의 API 키 입력
npm run dev
```
